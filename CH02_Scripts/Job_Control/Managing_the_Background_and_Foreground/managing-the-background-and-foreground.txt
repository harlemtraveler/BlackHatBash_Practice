Managing the Background and Foreground

Commands that you run in a terminal occupy that terminal until the command is finished.
These commands are considered foreground jobs.
In Chapter 1, we used the ampersand character (&) to send a command to the background.
This command then becomes a background job that allows us to unblock the execution of other commands.


# Run the "sleep" command and send it to the background using the ampersand (&)
sleep 100 &

# We can verify that the spawned process is running by using the "ps" command.
ps -ef | grep sleep

# We can use the "jobs" command to see what jobs are currently running.
jobs

# The output from the prior command above shows that the sleep command is in Running state and that its job ID is 1:
[1]+  Running                 sleep 100 &

# We can migrate the job from the background to the foreground by issuing the "fg" command and the job ID.
fg %1

# At this point, the sleep command is occupying the terminal, since it’s running in the foreground. 
# You can press CTRL-Z to suspend the process, which will produce the following output in the jobs table:
[1]+  Stopped                 sleep 100

# To send this job to the background again in a running state, use the bg command with the job ID.
bg %1

# In the prior command above, we supplied the job ID of 1.
# The output should like the following:
[1]+ sleep 100 &
