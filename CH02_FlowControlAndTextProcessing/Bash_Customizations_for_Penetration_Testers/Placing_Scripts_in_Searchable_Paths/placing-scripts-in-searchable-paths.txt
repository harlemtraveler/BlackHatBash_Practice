Placing Scripts in Searchable Paths

Bash searches for programs within directories defined by the PATH environment variable.
Commands such as ls are always available to you because system and user binaries are located in directories that are part of the PATH.


# To see your PATH, run this command:
echo $PATH


When you write a bash script, place it in a directory such as /usr/local/bin, which is part of the PATH.
If you don't do this, there're a few other options available:
- Call the script directly using the full path.
- Change directory to where the script lives and execute from there.
- Use aliases.
- Add paths to the PATH environment variable.

The benefit of placing the script in a searchable path is that you can simply call it by its name.
You don’t have to provide the full path or have the terminal be in the same directory.
